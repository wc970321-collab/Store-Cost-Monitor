# 新装店铺造价系统 (HG-Store-Cost-Monitor)

> **轻量级、无后端的商业空间建造与店铺调改造价监控系统。**  
> 专为零售、餐饮、连锁品牌新装或形象调改项目打造。基于纯前端解析本地 Excel 文件，实现店铺明细秒级呈现、模糊搜索、每平米单位造价换算以及跨工地项目造价的大盘平均值统计。

## 🎨 视觉与设计风格
本系统打破传统企业系统的沉闷感，采用年轻時尚的色块（Color-block）扁平化设计：
- **主题色**：太阳黄 `#FED961`（充满活力的主基调）
- **底色**：温和奶油白 `#FAF9F5`（提升阅读舒适度）
- **点缀色**：潮流粉 `#FF7597`（用于核心操作与重点数据高亮）
- **信息密度**：手风琴式折叠明细，支持高数据量下的“一屏式”总览

## 🚀 核心特性

- 🔒 **100% 本地隐私安全**
  所有 Excel 表格解析与计算均直接在用户的本地浏览器（Sandboxed）中完成，**数据绝不上传至任何服务器**，保障企业核心造价与商业机密安全。
- 🔍 **模糊搜索过滤**
  集成无刷新的高效过滤检索，支持输入店铺名称拼音或中文字词，在店铺繁多时实现秒级定位。
- 📦 **每平米单价与项数统计**
  自动提取店铺面积，换算单店及单项目的每平米单价。单位造价标签后实时显示该店已核算的项目项数（如 `(4项)`），使总造价包含的内容一目了然。
- 🔄 **智能多模刷新**
  - **手动刷新**：一键重载数据。传统导入模式下读取浏览器缓存，在自动同步下直接拉取本地磁盘最新快照。
  - **1小时自动刷新**：基于现代浏览器 *File System Access API*，授权一次后无需手动重复上传，页面保持打开即可在后台自动读取本地 Excel 修改。
- 📊 **智能大盘平均单位造价分析**
  自动剔除未填写/无面积数据的噪音干扰，按照 `(各工地单项每平米造价之和) ÷ 工地数` 的标准算法，输出各费用类别的全网平均单价，辅助新店铺建造预算决策。

## 📋 Excel 表格规范与填报指南
为保障系统能够正常、稳定地解析数据，Excel 表格的数据录入需遵循以下规范：

### 1. 标准列结构（请勿删除或调换顺序）
系统依赖关键字匹配列定位，模板必须包含以下列：
- **店名**：多行费用类别需合并单元格（如：`宜宾万象汇`、`云南曲靖店`）。
- **费用类别**：填入标准项目名称（如 `基装费用`、`空调改造`、`消防改造`、`货架`、`装修` 等）。
- **金额**：必须为**纯数字**（请勿带“元”或“万”字，未确定金额保持空白，勿写“无”、“待定”）。
- **面积**：对应店铺首行的面积列，必须为**纯数字**。

### 2. 标准项目名称禁止擅自修改
- 费用类别的标准字样是系统累计和分类求均值的唯一依据，严禁自行简写（例如将“空调改造”写成“空调”）、或随意加空格（例如写成“货  架”）。

## 💻 快速开始

### 方式 A：单机离线使用 (双击即用)
1. 下载本项目中的 `index.html`。
2. 双击在任何现代浏览器（推荐 Chrome 或 Edge）中打开。
3. 点击右上角 **“导入工程 Excel”**，导入符合规范的表格即可。

### 方式 B：局域网/异地实时查看 (服务器托管)
如果您希望本地修改 Excel 数据的同时，让其他同事可以在异地实时查看到最新数据：
1. 将 `index.html` 与保存的店铺造价数据表（**必须命名为 `data.xlsx`**）放在电脑的同一个文件夹内。
2. 启动一个本地轻型 Web 服务（例如在当前文件夹打开终端运行 `python -m http.server 8080`）。
3. 使用免费内网穿透工具（如 cpolar、樱花Frp）将您电脑的 `8080` 端口映射至公网。
4. 将生成的公网链接分享给同事：
   - 同事在异地浏览器打开该链接，网页会自动加载同级目录下的 `data.xlsx`。
   - 网页后台将**每 5 分钟自动从您的电脑获取一次最新保存的数据**，您在本地修改并覆盖保存 `data.xlsx` 后，同事的异地大屏会自动同步更新，无需购买域名和云服务器。
